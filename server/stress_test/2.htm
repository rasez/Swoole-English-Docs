Concurrent concurrent 100,000 TCP connection test

The code is in examples/c10k.php. This test starts 10 sub-processes and initiates a long connection to the Swoole TCP Server. Due to a single machine, the range of ip_local_port_range is 32000-60000. When running to 28000 long connections, the local port is not enough to continue connecting.

Swoole uses epoll as an event poll to maintain a large number of TCP connections. As long as the operating system has enough memory, it can always increase the maintained TCP long connection.

Swoole_server uses 70-100 bytes of memory per connection. Using data cache, such as EOF_CHECK/LENGTH_CHECK, may increase to 8K per connection.
