WebSocket

Swoole-1.7.9 adds built-in websocket server support, and can write an asynchronous non-blocking multi-process WebSocket server with a few lines of PHP code.

$server = new swoole_websocket_server("0.0.0.0", 9501);
 
$server->on('open', function (swoole_websocket_server $server, $request) {
    echo "server: handshake success with fd{$request->fd}\n";
});
 
$server->on('message', function (swoole_websocket_server $server, $frame) {
    echo "receive from {$frame->fd}:{$frame->data},opcode:{$frame->opcode},fin:{$frame->finish}\n";
    $server->push($frame->fd, "this is server");
});
 
$server->on('close', function ($ser, $fd) {
    echo "client {$fd} closed\n";
});
 
$server->start();

onRequest callback

Swoole_websocket_server inherits from swoole_http_server

  • Set the onRequest callback, the websocket server can also act as the http server at the same time.
  • The onRequest callback is not set, and the websocket server will return http 400an error page after receiving the http request.

Client

  • Chrome/Firefox/high version IE/Safari and other browsers have built-in JS language WebSocket client
  • WeChat applet development framework built-in WebSocket client
  • Asynchronous PHP programs can be used Swoole\Http\Clientas a WebSocket client
  • Apache/php-fpm or other synchronous blocking PHP programs can use the synchronous WebSocket client provided by swoole/framework
  • Non-WebSocket client cannot communicate with WebSocket server
