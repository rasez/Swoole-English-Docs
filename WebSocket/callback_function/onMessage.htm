onMessage

This function is called back when the server receives a data frame from the client.

function onMessage(swoole_server $server, swoole_websocket_frame $frame)
  • $frame is the swoole_websocket_frame object, which contains the data frame information sent by the client.
  • The onMessage callback must be set, the server will not start if it is not set.

swoole_websocket_frame

There are 4 attributes, respectively

  • $frame->fd, the client's socket id, which is needed to push data using $server->push
  • $frame->data, data content, can be text content or binary data, can be judged by the value of opcode
  • $frame->opcode, the OpCode type of WebSocket, you can refer to the WebSocket protocol standard document.
  • $frame->finish, indicating whether the data frame is complete, a WebSocket request may be split into multiple data frames for transmission.

$data If it is a text type, the encoding format must be UTF-8, which is specified by the WebSocket protocol.

OpCode and data types

  • OPCODE_TEXT_FRAME = 0x1, text data
  • OPCODE_BINARY_FRAME = 0x2 , binary data
