Reactor thread

The main process of Swoole is a multi-threaded program. There is a very important set of threads called Reactor threads. It is the thread that actually handles TCP connections and sends and receives data.

After the new connection of Accept, the main thread of swoole will assign the connection to a fixed Reactor thread, and this thread is responsible for listening to the socket. Reads the data as the socket is readable, performs protocol parsing, and delivers the request to the Worker process. Send data to the TCP client when the socket is writable.

The allocation is calculated as fd % serv->reactor_num
