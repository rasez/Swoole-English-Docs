Manager process

The worker/task process in swoole is managed by the Manager process Fork.

  • When the child process finishes running, the manager process is responsible for reclaiming this child process to avoid becoming a zombie process. And create a new child process
  • When the server is shut down, the manager process will send a signal to all child processes to notify the child process to close the service.
  • The server reload is, the manager process will close/restart the child process one by one.

Why is it not a Master process? The main reason is that the Master process is multi-threaded and cannot perform fork operations securely.
