Worker process

Swoole provides a complete process management mechanism, when the Worker process exits abnormally, such as a fatal error in PHP, a missed kill by another program, or a normal exit after reaching the max_request number. The main process will pull up the new Worker process again. The worker process can write code like normal apache+php or php-fpm. There is no need to write asynchronous callback code like Node.js.

Callback function in the main process :

  • onStart
  • onShutdown
  • onMasterConnect
  • onMasterClose
  • onTimer

Callback function within the worker process

  • onWorkerStart
  • onWorkerStop
  • onConnect
  • onClose
  • onReceive
  • onTimer
  • onFinish

Callback function in task_worker process

  • onTask
  • onWorkerStart

Management process callback function

  • onManagerStart
  • onManagerStop
