Swoole implementation

Swoole is written in pure C and does not depend on other third-party libraries.

  • Swoole does not use libevent, so there is no need to install libevent
  • Swoole does not rely on php's stream/sockets/pcntl/posix/sysvmsg extensions

Socket part

Swoole uses the underlying socket system call. See sys/socket.h

IO event loop

  • The main process's event loop uses select/poll, because there are only a few file descriptors in the main thread, use select/poll
  • Use epoll/kqueue in the reactor thread/worker process

There are a lot of people who use strace -p to see the main process of the swoole can only see the poll system call. The correct way to view is strace -f -p

Multi-process / multi-threading

  • Multi-process use fork() system call
  • Multithreading using the pthread thread library

EventFd

Eventfd is used in Swoole as a mechanism for thread/interprocess message notification.

Timerfd

Swoole uses timerfd to implement timers

SIgnalfd

Signalfd is used in swoole to shield and process signals. It can effectively avoid the problem that the thread/process is interrupted by the signal and the system calls restart. The reactor thread does not accept any signals in the main process.
