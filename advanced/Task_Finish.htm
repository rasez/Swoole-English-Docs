Use of the Task/Finish feature

The task module is used to do some asynchronous slow tasks, such as sending a broadcast in a webim, sending an email, and so on.

  • The task process must be synchronously blocked.
  • Task process support timer

Node.js If there are 100,000 connections, when you want to send a broadcast, it will cycle 100,000 times. At this time, the program can't do anything, can't accept new connections, and can't receive packets.

The swoole is different. After being thrown to the task process, the worker process can continue to process new data requests. After the task is completed, the worker process is notified asynchronously that it has completed the task.

Of course, the role of the task module is not only that, PHP database connection pool, asynchronous queues, etc., need to be further explored.
